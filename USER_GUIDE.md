# HVAC Calculator — Руководство пользователя

Расчёт тепловых нагрузок, вентиляции, систем дымоудаления, **ГВС**,
**энергопаспорта**, **подбора воздуховодов и труб отопления** и
**проверки точки росы** по данным из Revit
(СП 50.13330, СП 60.13330, СП 7.13130, СП 30.13330, СП 131.13330,
КМК Узбекистана).

---

## Новое в v3.7

Шесть новых функций (новая вкладка **«11. Расширения»** в GUI):

- **Точка росы** — проверка ограждений на риск конденсации (СП 50 Прил. Е).
  Для каждого внешнего ограждения считает τ_int, t_d, проверяет нормативный
  Δt. Раскрашивает плохие элементы красным/жёлтым.
- **ГВС (СП 30.13330)** — суточный/часовой расход горячей воды, мощность
  нагревателя, объём бака. Три стратегии: общая, по типу помещений, по зоне.
- **Энергопаспорт (СП 50 Прил. Г)** — годовое потребление тепла бин-методом
  по ГСОП, класс энергоэффективности (A++…E), отклонение от нормативного
  qh_у. Калибровка длительности отопит. сезона: ±5% на справочных городах
  СП 131.
- **Воздуховоды (СП 60 / АВОК)** — подбор Ø или AxB по рекомендованным
  скоростям, падение давления по Дарси-Вейсбаху, упрощённая сеть
  магистраль → ветка → терминал.
- **Трубы отопления** — подбор DN по СП 60, падение по Альтшулю,
  напор циркуляц. насоса в м.
- **PDF-отчёт «Пояснительная записка»** — 12 разделов, кириллица DejaVu,
  сводный документ для сдачи заказчику.

Плюс:
- 5 новых листов в Excel (всего 14): ГВС, Энергопаспорт, Точка росы,
  Воздуховоды, Трубы отопления.
- 98 новых unit-тестов; формулы Магнуса, Дарси, Альтшуля проверены
  вручную.
- В меню «Файл»: «Экспорт в Excel» и «Экспорт в PDF» вынесены наверх.

## Что делает программа

Один цикл расчёта на проект здания:

1. Загружает геометрию из 2 CSV-файлов (выгружаются из Revit через Dynamo)
2. Автоматически собирает каталог конструкций (стены, окна, витражи, полы…)
3. Считает теплопотери (СП 50.13330 + 4-зонный пол + подземные стены)
4. Считает теплопоступления с разделением Sensible / Latent (СП 60)
5. Считает воздухообмен по типам помещений (СП 60.13330, СП 44, СП 113)
6. Группирует помещения в зоны для подбора оборудования
7. Считает нагрузки от приточных установок (с учётом рекуперации)
8. Считает системы дымоудаления и подпора воздуха (КМК / СП 7.13130)
9. Экспортирует всё в Excel (11 листов) и обратно в Revit

**129 unit-тестов** покрывают всё ядро.

## Workflow — пошаговое использование

### 1. Подготовка данных в Revit

В Revit-проекте нужны **Spaces** (или Rooms) с заполненными параметрами.
Скрипт Dynamo `revit_dynamo_hvac_write_csv.py` (в архиве) выгружает:

- `spaces.csv` — список помещений с площадями и параметрами
- `thermal_all.csv` — все наружные ограждающие конструкции

**Важно для проекта Чорсу:**
- Curtain Walls автоматически распознаются как наружные стены
- Параметр **Room Bounding = True** должен стоять у наружных стен
- В CSV должны попасть все этажи (B1, B2, L01, L02, и т.д.)

### 2. Загрузка проекта (вкладка «1. Данные»)

- Указать пути к `spaces.csv` и `thermal_all.csv`
- Нажать «Загрузить CSV»
- Кнопка **«🔍 Проверить данные…»** — посмотреть предупреждения

### 3. Параметры проекта (вкладка «2. Параметры»)

Город → автоподстановка климата (85 городов СНГ).

**Для проекта Чорсу (Ташкент) поставить:**
- `t_out_heating` = −15°C (расчётная зимняя)
- `t_out_cooling` = 36°C (расчётная летняя)
- `w_out_summer_g_kg` = 7 (сухой климат — даст отрицательную скрытую инфильтрацию)
- `wwr_estimate` = 0.6 (для помещений где витраж не выгрузился)
- `solar_shading_factor` = 0.8 (внутренние жалюзи — норма для офисов)

### 4. Конструкции (вкладка «3. Конструкции»)

Программа автоматически собирает список всех типов стен/окон/витражей.
Задать U-значение и SHGC для каждого типа.

**Для современных зданий:**
- Стены: U = 0.4 Вт/(м²·К)
- Витраж Low-E: U = 1.8, **SHGC = 0.30** (важно для жаркого климата)
- Окна: U = 1.6, SHGC = 0.40

### 5. Помещения (вкладка «4. Помещения»)

- Программа автоопределит тип помещения по имени (Офис, Парковка, Санузел…)
- Двойной клик — редактирование одного помещения
- Ctrl+клик — массовое редактирование

**Что проверить:**
- Подвальные помещения помечены `has_floor_to_ground=True` (авто по уровню B1/B2)
- Верхний этаж помечен `has_roof=True`
- Угловые помещения подсвечены

### 6. Расчёт нагрузок (вкладка «5. Расчёт»)

Кнопка «▶ Рассчитать» → таблица результатов с разбивкой по статьям.

Колонки: Через ограждения, Солнце, Люди, Освещение, Оборудование,
Вентиляция, **Sensible**, **Latent**, ИТОГО, Уд. Вт/м².

### 7. Вентиляция (вкладка «6. Вентиляция»)

Кнопка «▶ Рассчитать вентиляцию» — Supply, Exhaust, Hood для каждого
помещения по нормам (СП 60 для общественных, СП 44 для гостиниц,
СП 113 для парковок).

Можно править вручную — двойной клик или массовое назначение.
Жёлтым подсвечены изменённые вручную (не пересчитываются автоматом).

### 8. Зоны и системы (вкладка «7. Зоны»)

- Авто-присвоение по префиксу номера (B01-* → «Блок B01»)
- Массовое назначение: выделить помещения → ввести имя зоны (например «П1») → применить
- Правая панель — сводка по зонам с **подсказкой типового размера** AHU/котла/чиллера

### 9. Оборудование (вкладка «8. Оборудование»)

Список систем с **индивидуальными параметрами**:
- **Приточные установки (AHU):** КПД рекуператора, температура подачи, влагосодержание
- **Котлы:** тип, t подачи, КПД, топливо
- **Чиллеры:** тип, t хладоносителя, COP, хладагент

Двойной клик → окно редактирования.

Кнопка «**Рассчитать нагрузки от AHU**» — Q калорифера и Q охладителя
для каждой приточной установки с учётом рекуперации.

**Без рекуперации зимой:** Q калориферов ≈ 50–60% всей тепловой нагрузки.
Поставьте рекуператоры на офисные/гостиничные AHU.

### 10. Дымоудаление (вкладка «9. Дымоудаление»)

- **▶ Авто-присвоение** — создаёт СДУ для парковок/складов/коридоров и СПВ для лестниц/лифтов
- Радио «Сценарий пожара»: один пожар в одной зоне (стандарт) или все зоны (запас)
- Радио «Показать»: СДУ или СПВ
- Двойной клик → правка норматива, площади зоны, давления, огнестойкости

### 11. Графики (вкладка «10. Графики»)

8 типов графиков: по уровням, по типам, разбивка нагрузок, heat map по
этажам, распределение U-значений, и др.

### 12. Сохранение / экспорт

- **Меню «Файл → Сохранить проект»** — JSON со всеми настройками
- **Меню «Файл → Экспорт в Excel»** — 11 листов:
  - Параметры, Конструкции, Теплопотери, Теплопоступления,
    Ограждения, Вентиляция, Сводка по уровням,
    Зоны и системы, Системы оборудования, Дымоудаление, Проверки
- **Меню «Файл → Экспорт в Revit»** — CSV для обратной записи нагрузок в Spaces

## Известные ограничения

1. **WWR подменяется реальным стеклом из Revit** — если у вас выгружены
   витражи, WWR на стены не накладывается (исправление double-counting)

2. **Ориентация фасадов** — берётся из азимутов в CSV. Если Dynamo не
   выгрузил азимуты, используется средний коэффициент 0.65

3. **Скрытая инфильтрация в сухом климате отрицательна** — это правильно
   физически (наружный воздух осушает помещения)

4. **63 СПВ на проект — много** — это потому что в Revit каждый лифт
   отдельный Space. На реальном здании их можно объединить в 5-10 систем
   через массовое переименование на вкладке «9. Дымоудаление»

5. **Гидравлический расчёт не делается** — только подбор размера установок.
   Конкретный модель оборудования и трассировка — в другой программе.

## Расширение функционала

Программа модульная. Чтобы добавить новую методику расчёта:

```python
# hvac/engine/my_method.py
from hvac.engine.base import CalculationEngine, register_engine

@register_engine
class MyEngine(CalculationEngine):
    @property
    def name(self):
        return "ASHRAE 90.1"

    def heat_loss(self, space, project):
        # ... ваша логика
        return {"ИТОГО": ...}

    def heat_gain(self, space, project):
        return {...}
```

После этого новая методика доступна в выпадающем списке параметров.
Аналогично для графиков (`@register_chart`) и движков вентиляции
(`@register_ventilation_engine`).
